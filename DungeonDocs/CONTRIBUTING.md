# Contributing Guidelines

## 1. Getting Started
Ensure you have CMake, a C++17 compatible compiler, and the necessary SDKs. Read `ENGINE_CONSTITUTION.md` before writing code.

## 2. Branching
Create feature branches from `main`. Use descriptive names (e.g., `feature/render-pipeline`, `fix/memory-leak`).

## 3. Pull Requests
* Provide a clear title and description.
* Ensure the Definition of Done is met.
* All CI checks (Catch2 tests, formatting, warnings) must pass.

## 4. Code Review
Be respectful and constructive. Focus on architecture, performance, and readability.

## 5. Reporting Issues
Use the issue tracker. Provide reproduction steps, expected behavior, and actual behavior.
